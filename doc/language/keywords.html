Keywords

List of keywords known by the as2js compiler and their basic usage. The system also supports a certain number of attributes which look like keywords but are weak, meaning that you can still use those identifiers as function names or variable names. These are marked with a w as in arrayw.

  • abstract -- mark a function as abstract, usually in interfaces
  • arrayw -- function attribute (but I don't remember for what at this point...)
  • as -- convert a variable type to another
  • autobreakw -- to mark a switch statement as having breaks in all cases, automatically
  • boolean -- internal type, use Boolean as the object
  • break -- break loops, exit a switch, support one label
  • byte -- internal type, use Integer as the object
  • case -- define a case in a switch
  • catch -- handle an exception in a try/catch/finally block
  • char -- internal type, use Integer as the object
  • class -- declare an object functions and variables
  • const -- define a constant variable
  • constructorw -- to mark a function in a class as a constructor, whatever its name
  • continue -- repeat a loop, support one label
  • debugger -- open the debugger and stop (JavaScript breakpoint)
  • default -- used in a switch statement to define a default case
  • delete -- delete an object
  • deprecatedw -- to mark a class, function, variable as deprecated (should not be used anymore)
  • do -- start a while() loop repeated at least once
  • double -- internal type, use Double as the object
  • dynamicw -- to mark a class as accepting additional definitions at runtime
  • else -- preceeds the block of statements to execute when an if condition is false; also used with the function require keyword
  • enum -- declare a set of variables
  • ensure -- expressions that must be true before exiting a function
  • enumerablew -- to mark a function or variable member as visible when enumerating the object members
  • export -- explicit export of package declarations
  • extends -- a class or interface that extend another class or interface
  • false -- the boolean false
  • final -- an attribute to mark a function as final (cannot be overloaded anymore) or a variable to be assigned at most once
  • finally -- a block of code to always execute in a try/catch/finally statement
  • float -- internal type, use Double as the object
  • for -- start a for or for each loop
  • foreachw -- to mark a switch statement as a foreach statement: i.e. all case blocks that match are ran
  • function -- declare a function, like in C/C++ a function can be declared without a body and the body defined later
  • goto -- continue processing at the specified label
  • if -- conditional execution of statements
  • implements -- a class can implement an interface
  • import -- to import a package in your code
  • in -- test whether a value is defined in an array and for each keyword
  • inline -- whether a function or variable is considered inline
  • instanceof -- whether an object is an instance of a specific type
  • interface -- define an interface, a class with only abstract functions and constant variable members
  • internalw -- mark a class or function as intrinsic (defined internally in JavaScript)
  • invariant -- define a contract, a set of expressions that need to remain true while a object of the class exists
  • is -- check whether an object is of a given type
  • Infinity -- the positive infinity value, directly compiled as a FLOAT64
  • long -- internal type, use Integer as the object
  • namespace -- declare a namespace, use a namespace
  • native -- mark a package, class, function as native (defined in the JavaScript language)
  • new -- create an object, must be followed by the name of a class and optional parameters
  • nobreakw -- explicitly mark a switch statement as allowing one case to continue in the next case
  • null -- the value null, this is a JavaScript value
  • NaN -- the Not-a-Number value, directly compiled as a FLOAT64
  • package -- define a package, packages can be imported
  • private -- define a class, function, variable as private: not accessible outside of its package or class
  • protected -- define a class, function, variable as protected: not accessible outside of its package or class or derivation thereof
  • public -- define a class, function, variable as public: accessible by anyone
  • require -- define a contract for a function, also supports require else
  • return -- exit a function with an optional value
  • short -- internal type, use Integer as the object
  • static -- a static function, which can be called without an instance of the object
  • super -- access to the super class members (the class you extend from)
  • switch -- conditionally process one case or more statements
  • synchronized -- only one thread processes the following statements
  • then -- used in the ensure then contract definition
  • this -- access to this object
  • throw -- raise an exception
  • throws -- list exceptions that a function may raise
  • transient -- mark a variable member as having transient data (not to be saved when serialized)
  • true -- the boolean true
  • try -- start an exception protected block, must be followed by one catch or finally statement
  • typeof -- return the type of an object as a string
  • undefined -- the undefined value, we define it as a keyword to avoid overloading bugs
  • unsafew -- mark a function as unsafe for use (i.e. eval() is marked as such)
  • unusedw -- mark a function as unusable (possibly was marked as deprecated in previous versions)
  • use -- pragma definitions and use namespace definitions
  • var -- declare a variable
  • virtualw -- mark a function as virtual, meaning it can be overloaded
  • void -- internal type, use Void as the object
  • volatile -- mark a variable as volatile (eventually changing under your feet)
  • while -- create a do or while loop
  • with -- create a block that has access to variable members of an object or to define the operator to use in a switch statement
  • yield -- stop processing, a poor man's way of allowing long computations to happen interactively
  • __FILE__ -- the name of the file being processed
  • __LINE__ -- the line number on which __LINE__ is defined
